📊 Simulación de Negocio

Manual completo de usuario y configuración técnica para el módulo de Simulación de Negocio del sistema ARPI. Cubre el flujo de trabajo desde la creación hasta la conversión a negocio, la generación de reportes y todos los parámetros configurables.

Versión 2025  |  Módulo: newsimulator  |  Backend: TarpSimulaciones*  |  Fecha: abril 2026
📌

1. Introducción

El módulo de Simulación de Negocio permite a los asesores comerciales de ARPI modelar escenarios de inversión para un prospecto (inversionista). A partir de la simulación se genera un PDF personalizado, se envía por correo electrónico y, cuando el prospecto decide avanzar, la simulación se convierte automáticamente en un negocio formal dentro del sistema.

👤

Perfilación del inversor

Se capturan datos personales, segmento y perfil de riesgo del prospecto.

🎯

Propósitos de inversión

El asesor define los objetivos financieros del cliente (educación, retiro, etc.).

💰

Fuentes de ahorro

Registro de los ingresos y ahorros disponibles del prospecto.

📦

Mejor propuesta (Paquete)

El sistema sugiere el paquete de proyectos más adecuado al perfil.

📅

Forma de pago

Se define el plan de cuotas y pagos para cada proyecto del paquete.

🔁

Conversión a negocio

Con un clic la simulación genera negocios reales y su plan de pagos.

🔐

2. Acceso al módulo

El módulo de simulación está disponible en el menú lateral de ARPI bajo la opción Simulaciones. La ruta de acceso en el navegador es:

/simulator/simulaciones    ← Listado de simulaciones
/simulator/simulator        ← Formulario nueva / editar
ℹ️
Para crear o editar simulaciones el usuario debe tener sesión activa en AWS Cognito. El nombre y documento del asesor se obtienen automáticamente de los atributos del usuario autenticado (name, family_name, custom:document).
📋

4. Crear una nueva simulación

Para crear una simulación haga clic en el botón "Iniciar simulación" en la parte superior del listado. El sistema lo llevará al formulario multisección organizado en pestañas (tabs). Cada pestaña se habilita progresivamente conforme se completan los datos requeridos.

1

Pestaña 1 — Datos del Inversionista

Seleccione el tipo de persona (Natural o Jurídica). Ingrese nombre, apellidos, teléfono y correo. Use el botón Buscar para consultar si el contacto ya existe en el sistema (Bitrix). Si no existe, se abre el formulario de creación de contacto. Una vez localizado el contacto, la pestaña de Perfilación se activa.

2

Pestaña 2 — Perfilación

Se presentan las preguntas de perfil del inversionista obtenidas de la maestra PERFILACIONES. Todas las preguntas deben ser respondidas para habilitar la siguiente pestaña. Los tipos de respuesta son: Lista (combo) u Objetos (selección de ítem).

3

Pestaña 3 — Propósitos

Seleccione uno o más propósitos de inversión del listado. Para cada propósito se captura: Tiempo para cumplirlo, Valor aproximado y Cantidad de unidades. Al completar todos los campos, se habilita la siguiente pestaña.

4

Pestaña 4 — Fuentes de Ahorro

Se registran los ingresos y ahorros periódicos del prospecto. Para cada fuente se define: Valor y Frecuencia (mensual, trimestral, etc.). Los códigos de las fuentes seleccionadas se almacenan en jsfuentesdeahorro y en el campo fahorros para integración con Bitrix24.

5

Pestaña 5 — Mejor Propuesta (Paquete)

El sistema muestra los paquetes de proyectos disponibles según el perfil del cliente. El asesor selecciona el paquete más adecuado. Cada paquete incluye uno o más proyectos con sus valores, número de ARPIs disponibles y condiciones.

6

Pestaña 6 — Forma de Pago

Se define el plan de cuotas para cada proyecto del paquete. Se calculan el total de inversión (nmtotalinversion), el ajuste de forma de pago (nmajusteformapago) y el total a pagar (nmtotalapagar). El plan se almacena en formato JSON en el campo jsformadepago.

7

Guardar la simulación

Al hacer clic en Guardar, el sistema solicita confirmación. Si se confirma, se llama al API POST /api/TarpSimulaciones. El backend registra la simulación, crea el deal en Bitrix24 e integra los IDs obtenidos en el JSON de proyectos. Al finalizar, el sistema redirige al listado de simulaciones.

💡
Datos adicionales en la primera pestaña: también puede registrar la Fecha de reunión, el canal por el cual el prospecto conoció a ARPI (Cómo nos conoció) y el nombre de la persona que lo refirió (Referido por).
✏️

5. Editar una simulación existente

Desde el listado, haga clic en el ícono edit (✏️) de la fila deseada. El sistema carga el formulario con todos los datos precargados:

  • Datos del inversionista (tipo persona natural/jurídica)
  • Perfilación previamente registrada
  • Propósitos seleccionados con sus valores
  • Fuentes de ahorro con valores y frecuencias
  • Paquete seleccionado con los valores de proyectos
  • Forma de pago (cuotas por proyecto)
⚠️
Al guardar una simulación editada el sistema guarda una nueva simulación vinculada a la original mediante el campo nmsimulacionOrigen. La simulación original no se modifica, permitiendo trazabilidad histórica completa.
🔁

6. Convertir simulación a negocio

Esta es la función más importante del módulo. Al hacer clic en el botón ▶️ (skip_next), el sistema:

1

Selección del vendedor por proyecto

Se abre un modal donde se selecciona el inversionista-vendedor para cada proyecto incluido en la simulación. Esto determina de quién compra cada ARPI el nuevo inversor.

2

Confirmación

El sistema solicita confirmación antes de proceder: "¿Desea convertir la simulación en negocio?"

3

Validaciones en el backend

Se valida que el email del prospecto exista como inversionista y que el documento del consultor exista como vendedor. Si alguna validación falla, se muestra el error específico sin crear ningún negocio.

4

Creación de negocios

Por cada proyecto del paquete con cantidad mayor a 0 y sin negocio previo (cdunicnegocio vacío), se crea un registro en tarp_negocios con estado C y operación VENTA.

5

Plan de pagos

Para cada negocio creado, el sistema genera automáticamente el plan de pagos en tarp_planpagos_negociacion a partir del JSON de forma de pago de la simulación.

6

Actualización de la simulación

El estado de la simulación se cambia a F (Finalizada) y el JSON de valores de proyectos se actualiza con los cdunicnegocio de cada negocio creado.

✅
Si el proceso es exitoso, se muestra "Negocio generado correctamente". Si algún proyecto ya tenía negocio creado, el mensaje advierte sobre ello sin considerarlo un error fatal.
📧

7. Generar PDF y enviar por correo

Generar PDF de simulación

Desde el listado, use el botón de descarga (⬇️ → seleccione plantilla "Simulador") o desde el interior del formulario. El PDF se genera con el reporte Jasper reporteSimulacionPDF.jasper usando el parámetro p_nmsimulacion.

Endpoint:

GET /api/TarpSimulaciones/getSimulacionPdf/{id}

Retorna un arreglo de bytes (Base64) con el PDF listo para descarga.

Envío por correo electrónico

El sistema envía un correo al email del inversionista adjuntando el PDF generado. El correo usa la plantilla Thymeleaf simulacion.html con los siguientes datos dinámicos:

Variable ThymeleafDescripciónOrigen
${fecha}Fecha actual en español (Colombia)Generado en backend
${nmsimulacion}Número de la simulacióntarp_simulaciones.nmsimulacion
${nombre}Nombre del prospectoNatural: dsnombres + dsapellidos / Jurídica: razonSocialPerJuridica
${email}Correo del prospectoNatural: dsemail / Jurídica: emailPerJuridica
${telefono}Teléfono del prospectoNatural: dstelefono / Jurídica: telPerJuridica

Endpoint:

GET /api/TarpSimulaciones/sendMailSimulacion/{id}
ℹ️
El asunto del correo es fijo: "SIMULACION COMPRA DE ARPIS". El correo siempre se envía a simulacion.getDsemail() independientemente del tipo de inversionista.
📄

8. Descargar plantillas

Existen dos tipos de plantillas descargables desde el listado:

📁

Plantilla general del simulador

Genera documentos .docx del grupo "Simulador". Se abre un modal para elegir la plantilla deseada. El API invocado es POST /api/TarpReportes/SimuladorPlantilla.

📂

Plantilla por proyecto

Genera un .docx específico por cada proyecto de la simulación usando el endpoint de plantilla de proyecto. Se descarga automáticamente un archivo por proyecto.

📊

9. Exportar a Excel

En la parte inferior del listado hay un botón XLS que exporta las simulaciones visibles (según el filtro de estado activo) a un archivo Excel (.xlsx). Las columnas exportadas son:

  • N°, Nombre, Documento, Email, Asesor, Paquete
  • Fecha de creación, N° Simulación Origen
  • Total inversión, Aspiración renta, Estado
💡
El nombre del archivo Excel varía según el estado activo: Simulaciones_Activas.xlsx, Simulaciones_Desistidas.xlsx o Simulaciones_Aprobadas.xlsx.
🗑️

10. Declinar simulación

El botón delete (🗑️) solo es visible para simulaciones en estado A (Activa). Al hacer clic, el sistema cambia el estado de la simulación a D (Desistida) llamando al endpoint POST /api/TarpSimulaciones/saveOrUpdate.

⚠️
Esta acción no elimina los datos de la simulación de la base de datos. Únicamente cambia el campo cdestado a D. La simulación sigue siendo consultable filtrando por estado "Desistida".

Consulta SQL principal (SELECT usado por el DAO)

El SELECT del DAO une la tabla de simulaciones con tarp_paquetes y tarp_contactos para obtener los datos actualizados del contacto Bitrix:

SELECT s.nmsimulacion, s.dtfechacreacion, s.cddocumento,
  -- Teléfono: usa Bitrix si hay contacto, sino el valor guardado
  if(s.cdunicontacto = '', dstelefono,
     IF(c.jsdata ->>'$.PHONE[0].VALUE' = 'null', ' ',
        jsdata ->>'$.PHONE[0].VALUE')) as dstelefono,
  -- Email: usa Bitrix si hay contacto
  if(s.cdunicontacto = '', dsemail,
     IF(c.jsdata ->>'$.EMAIL[0].VALUE' = 'null', ' ',
        jsdata ->>'$.EMAIL[0].VALUE')) as dsemail,
  s.dsconsultor, s.jsperfilacion, s.jspropositos,
  s.jsfuentesdeahorro, s.jsvaloresproyectos, s.jsformadepago,
  -- Nombre: usa Bitrix si hay contacto
  if(s.cdunicontacto = '',
     concat(s.dsnombres,' ', ifnull(s.dsapellidos,'')),
     concat(c.jsdata ->>'$.NAME',' ',
     IF(c.jsdata ->>'$.SECOND_NAME'='null',' ',
        jsdata ->>'$.SECOND_NAME'))) as dsnombres,
  (if(tipoInversionista = 1, nitPerJuridica, cddocumento)) as dsdocumento,
  pq.dspaquete
FROM tarp_simulaciones s
INNER JOIN tarp_paquetes pq ON s.nmpaquete = pq.nmpaquete
LEFT JOIN  tarp_contactos c  ON s.cdunicontacto = c.cdunicontacto
WHERE s.cdestado = 'A'
ORDER BY s.nmsimulacion DESC;

15. Configuración del correo electrónico

Plantilla de correo

Ubicación: arpi-back/src/main/resources/templates/simulacion.html

Esta es la plantilla Thymeleaf que define el cuerpo del correo. Para modificar el texto del email, edite este archivo. Las variables dinámicas disponibles son: fecha, nmsimulacion, nombre, email, telefono.

Configuración del servidor SMTP

La configuración del servidor de correo se encuentra en los archivos de propiedades de Spring Boot:

# application.properties / application-dev.properties
spring.mail.host=smtp.tudominio.com
spring.mail.port=587
spring.mail.username=correo@tudominio.com
spring.mail.password=tu_contraseña
spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true

Ubicación del archivo: arpi-back/src/main/resources/application.properties (o application-dev.properties para el entorno de desarrollo).

Clase de envío de correos

Clase: GestionCorreos.java — Método: sendMailGeneric()

Este método recibe el HTML procesado, el asunto, el destinatario, el adjunto y el nombre del archivo adjunto. Para la simulación, el destinatario siempre es simulacion.getDsemail() (email persona natural).

⚠️
Nota importante: Existe un error de comparación de String en el código actual. En el método sendMailSimulacion la comparación tipoInv == "1" siempre es false en Java (debe usarse .equals("1")). Por ello, el correo siempre usa los datos de persona natural (dsnombres, dsemail), independientemente del tipo de inversionista. Esto debe corregirse si se requiere soportar personas jurídicas.
🔗

16. Integración con Bitrix24

Al crear una simulación (endpoint POST /api/TarpSimulaciones), el backend crea automáticamente un Deal (negocio) en Bitrix24 para cada proyecto incluido. El flujo es:

en BD

Webhooks configurados

ConstanteCuándo se usaDescripción
Constantes.WEBHOOKSIMULACION[0] Cuando iddealbitrix es vacío o 0 Crea un nuevo Deal en Bitrix24 para el proyecto
Constantes.WEBHOOKSIMULACION[1] Cuando ya existe un iddealbitrix Actualiza el Deal existente en Bitrix24
ℹ️
Los webhooks se consultan en la tabla tarp_integraciones_metodos usando el valor de la constante como código maestro. Para cambiar la URL del webhook, modifique el registro correspondiente en esa tabla.

Campo iddealbitrix en jsvaloresproyectos

Cada elemento del JSON de proyectos almacena el ID del Deal de Bitrix24 correspondiente. Este ID se usa posteriormente al convertir la simulación a negocio para vincular ambos sistemas.

{

21. Dónde configurar un Proyecto para Simulaciones

Para que un proyecto aparezca correctamente en el simulador y muestre proyecciones, renta esperada y cálculos financieros correctos, deben configurarse dos lugares:

⚙️

1. Engranaje del Proyecto

Botón settings ⚙️ en la tabla de Proyectos. Abre los parámetros internos del proyecto que afectan los cálculos del simulador.

📦

2. Paquete (módulo Paquetes)

El proyecto debe estar asociado a un Paquete activo. En el paquete se definen el valor del ARPI, cantidades, plazo y porcentaje de renta para esa combinación proyecto+paquete.

⚠️
Sin ambas configuraciones el simulador no funcionará correctamente. El proyecto puede existir pero si no está en un paquete activo, el asesor no lo verá al simular. Si está en el paquete pero el engranaje tiene valores en 0, los cálculos de renta y proyección serán incorrectos.
🔧

22. El Engranaje ⚙️ — Parámetros internos del Proyecto

En el módulo Proyectos, en la tabla de proyectos, cada fila tiene un botón con el ícono ⚙️ settings (tooltip: "Actualizar Parámetros"). Al hacer clic se abre un modal con la tabla de parámetros internos del proyecto (tarp_configuracion_proyecto).

📍 Ruta en ARPI:

Menú lateral → Proyectos → Fila del proyecto → Botón ⚙️ (settings)

Cómo editar un parámetro del engranaje

1

Abrir el modal de configuración

Haga clic en el botón ⚙️ de la fila del proyecto que desea configurar.

2

Localizar el parámetro

La tabla muestra dos columnas: Parámetro (código) y Valor (campo editable directamente en la tabla).

3

Modificar el valor

Haga clic en el campo Valor del parámetro que necesita cambiar y escriba el nuevo valor.

4

Guardar

Haga clic en el botón 🔄 update de la misma fila. El cambio se guarda inmediatamente en la base de datos.

Parámetro (cdparametro)Tipo de ValorQué hace en la SimulaciónEjemplo
PLANTILLA-SIMULACIONTexto (ID)Identifica la plantilla Word (.docx) que se genera al descargar la plantilla de simulación de este proyecto desde el listadoSIMUL-PRY01

Parámetros adicionales para cálculo de renta (críticos para el simulador)

Estos parámetros también se leen desde tarp_configuracion_proyecto y son los más importantes para el simulador. Si no están configurados correctamente la renta aparecerá en $0 y la gráfica de proyección estará vacía.

ParámetroTipoQué calcula en el SimuladorEjemplo
VALOR-RENTA-ESPERADA Número (COP) ⭐ El más importante. Renta mensual esperada por ARPI. Base del cálculo: RentaTotal = VALOR-RENTA-ESPERADA × NroARPIs × mesesRestantesAño. Si está en 0, la renta del cliente aparece $0 en la simulación. 250000 (COP por ARPI/mes)
CANT_MESES_PROYECCION Número entero (años) ⭐ El segundo más importante. Número de años proyectados hacia el futuro en la gráfica de proyección. Si está en 0, no se muestra ninguna gráfica. 10 (10 años de proyección)
PUNTOS-SPREAD Número entero Puntos adicionales sobre el IPC para la valorización del activo. Afecta la curva de valor comercial proyectado. Fórmula: ValorComercialSig = ValorComercial × (IPC + Spread + 1) / 100 2
PORCENTAJE-RENTA-ESPERADA Número decimal (%) Porcentaje de renta esperada aplicado sobre el valor del ARPI. Complementa el cálculo de VALOR-RENTA-ESPERADA 5.5
ℹ️
Estos parámetros (VALOR-RENTA-ESPERADA, CANT_MESES_PROYECCION, PUNTOS-SPREAD, PORCENTAJE-RENTA-ESPERADA) se crean y editan también desde el modal del engranaje ⚙️ si ya existen en la tabla, o se insertan directamente en base de datos si aún no existen para ese proyecto.

SQL de consulta y actualización directa

-- Ver todos los parámetros de un proyecto
SELECT cdparametro, dsvalor_parametro
FROM   tarp_configuracion_proyecto
WHERE  cdunicproyect = 'TU-CODIGO-PROYECTO'
ORDER BY cdparametro;

-- Actualizar la renta esperada de un proyecto
UPDATE tarp_configuracion_proyecto
SET    dsvalor_parametro = '250000'
WHERE  cdparametro   = 'VALOR-RENTA-ESPERADA'
  AND  cdunicproyect = 'TU-CODIGO-PROYECTO';

-- Insertar el parámetro si no existe
INSERT INTO tarp_configuracion_proyecto
  (cduniconfpro, cdparametro, dsvalor_parametro, cdunicproyect)
SELECT UUID(), 'VALOR-RENTA-ESPERADA', '250000', 'TU-CODIGO-PROYECTO'
WHERE NOT EXISTS (
  SELECT 1 FROM tarp_configuracion_proyecto
  WHERE cdparametro = 'VALOR-RENTA-ESPERADA'
    AND cdunicproyect = 'TU-CODIGO-PROYECTO'
);

-- Ver diagnóstico completo: todos los proyectos con sus parámetros clave
SELECT
  pr.dsproyecto,
  MAX(CASE WHEN c.cdparametro = 'VALOR-RENTA-ESPERADA'    THEN c.dsvalor_parametro END) AS renta_esperada,
  MAX(CASE WHEN c.cdparametro = 'CANT_MESES_PROYECCION'     THEN c.dsvalor_parametro END) AS meses_proyeccion,
  MAX(CASE WHEN c.cdparametro = 'PUNTOS-SPREAD'             THEN c.dsvalor_parametro END) AS spread,
  MAX(CASE WHEN c.cdparametro = 'FECHA-INICIO-RENTA'        THEN c.dsvalor_parametro END) AS inicio_renta,
  MAX(CASE WHEN c.cdparametro = 'VALOR-MINIMO-INVERSION'    THEN c.dsvalor_parametro END) AS min_inversion,
  MAX(CASE WHEN c.cdparametro = 'PLANTILLA-SIMULACION'      THEN c.dsvalor_parametro END) AS plantilla_sim
FROM   tarp_proyectos pr
LEFT JOIN tarp_configuracion_proyecto c ON pr.cdunicproyect = c.cdunicproyect
GROUP BY pr.dsproyecto
ORDER BY pr.dsproyecto;

23. Módulo Paquetes — Conectar Proyectos con el Simulador

El Paquete es el puente entre los proyectos y el simulador. Cuando el asesor llega a la pestaña "Mejor Propuesta", el sistema muestra los paquetes activos que coinciden con el perfil del cliente. Si el proyecto no está en un paquete activo y vigente, no aparecerá en ninguna simulación.

📍 Ruta en ARPI:

Menú lateral → Paquetes → Botón + (Agregar) o seleccionar uno existente para editar

Campos del Paquete que afectan directamente la Simulación

Campo del PaqueteEfecto en SimulaciónDescripción

dscaracteristicasSe muestra al asesor en el tab Mejor PropuestaTexto de características del paquete para presentar al cliente
dsobjetivoAyuda al asesor a elegir el paquete según el propósito del clienteObjetivo del paquete
nmporccomisionNo visible al cliente, usado internamentePorcentaje de comisión del asesor

cdestadoSolo los paquetes con estado activo aparecen en el simuladorEstado del paquete (A = activo)

CampoImpacto en SimulaciónDescripciónEjemplo

Paso a paso: agregar un proyecto a un paquete

1

Ir al módulo Paquetes

Menú lateral → Paquetes. Seleccione el paquete a editar o cree uno nuevo.

2

Clic en "Agregar proyecto"

Aparece un nuevo bloque de campos para el proyecto.

3

Seleccionar el proyecto

En el selector desplegable "Proyectos", elija el proyecto. Solo aparecen los proyectos registrados en el módulo Proyectos.

4

Llenar todos los campos del proyecto en el paquete

Complete: Valor del ARPI, Cantidad mínima ARPIs, Plazo en meses, Porcentaje de renta (%), Cantidad ARPIs disponibles y Porcentaje financiación (%).

5

Verificar fechas del paquete

La Fecha inicial y Fecha final del paquete deben cubrir el período de ventas. La fecha actual debe estar entre ellas.

6

Guardar

Clic en el botón 💾 (save). El sistema guarda el paquete y la relación con el proyecto.

SQL de diagnóstico de Paquetes y Proyectos

-- Ver todos los paquetes activos y vigentes con sus proyectos
SELECT
  pk.nmpaquete, pk.cdpaquete, pk.dspaquete,
  pk.dtfechainicial, pk.dtfechafinal, pk.cdestado,
  pr.dsproyecto,
  pp.nmvalorarpi, pp.nmcantminarpis,
  pp.nmplazo, pp.nmvalorrenta,
  pp.nmcantarpisdisponibles
FROM   tarp_paquetes pk
JOIN   tarp_proyectos_paquete pp ON pk.nmpaquete = pp.nmpaquete
JOIN   tarp_proyectos pr ON pp.cdunicproyect = pr.cdunicproyect
WHERE  pk.cdestado = 'A'
  AND  CURDATE() BETWEEN pk.dtfechainicial AND pk.dtfechafinal
ORDER BY pk.nmpaquete;

-- Ver paquetes vencidos (no aparecen en el simulador)
SELECT nmpaquete, cdpaquete, dspaquete, dtfechainicial, dtfechafinal
FROM   tarp_paquetes
WHERE  CURDATE() > dtfechafinal OR cdestado != 'A'
ORDER BY dtfechafinal DESC;

-- Actualizar cantidad disponible de ARPIs en un paquete
UPDATE tarp_proyectos_paquete
SET    nmcantarpisdisponibles = 45
WHERE  nmpaquete      = 1
  AND  cdunicproyect = 'PRY001';

20. Errores frecuentes y soluciones

Error / MensajeCausaSolución
"El contacto no se encontró" El cdunicontacto no existe en tarp_contactos Verificar que el contacto esté sincronizado desde Bitrix24. Usar la búsqueda de contactos en el formulario.
"El correo electrónico no se encuentra creado como inversionista" Al convertir a negocio, el email del prospecto no existe en tarp_inversionistas Crear primero el inversionista en el módulo de Inversionistas con ese email exacto.
"El número de documento no se encuentra creado como vendedor" El documento del consultor (dsconsultordocumento) no existe en tarp_vendedores Verificar que el asesor esté registrado como vendedor. El documento se obtiene de AWS Cognito (custom:document).
"El paquete no existe o no se encuentra activo" El nmpaquete no existe en tarp_paquetes Verificar que el paquete seleccionado esté activo en el módulo de Paquetes.
"Error al extraer valores de la simulación" El JSON jsvaloresproyectos o jsformadepago está malformado Revisar el JSON almacenado en BD. Puede deberse a caracteres especiales no escapados.
PDF no se genera / Error FactoryReportException El archivo reporteSimulacionPDF.jasper no existe o la BD no retorna datos Verificar que el .jasper esté en src/main/resources/ y que la simulación tenga datos completos.
Correo no llega Configuración SMTP incorrecta o error en plantilla Verificar application.properties SMTP. Revisar logs del backend (arpi.log) para el error específico.
La pestaña de Perfilación no se habilita El contacto no se encontró o no se seleccionó en el buscador Usar el botón "Buscar" e inmediatamente seleccionar el contacto. El sistema activa la pestaña al confirmar la selección.
"El proyecto YA se creó como negocio" El campo cdunicnegocio en jsvaloresproyectos ya tiene valor Situación normal cuando ya se convirtió ese proyecto. No es un error, es un aviso informativo.
